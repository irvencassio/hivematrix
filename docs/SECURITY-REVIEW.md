# Security Review — 2026-06-11

Final review pass over the security-sensitive surfaces (token handling, the
daemon + helper HTTP surfaces, the AppleScript allowlist).

**Update (2026-06-11, remediation):** F1–F4 and F6 have been **implemented and
verified live** (Irv green-lit the fixes); F5 was already applied; F7 remains an
accepted INFO. See per-finding status below.

Context that bounds severity: single-user, single-Mac appliance; both HTTP
surfaces bind **loopback only** (not network-reachable).

---

## F1 — Daemon HTTP API is unauthenticated with wildcard CORS  · HIGH · FIXED

`src/daemon/server.ts` previously set `Access-Control-Allow-Origin: *` with no
auth, so any web page could issue cross-origin requests to `:3747`.

**Fixed:** the wildcard CORS header is removed (the console is same-origin; the
Tauri shell navigates to the daemon origin). All routes except liveness/page
loads (`GET /health`, `/`, `/console`) now require a shared-secret bearer token
(`~/.hivematrix/auth-token`, mode 600, via `src/lib/auth/token.ts`). The token
is injected into the same-origin console HTML at serve time, so the browser
console works but a cross-origin page cannot read it (and thus cannot forge
authenticated requests). SSE passes the token as a query param.
**Verified live:** `/metrics` no-token → 401, with token → 200, `/health` → 200,
console carries the injected token, soak unaffected (56 runs / 0 failed).

## F2 — DesktopBee helper API is unauthenticated  · HIGH · FIXED

The helper bound loopback but had no caller auth, and approval tiers were
enforced only client-side.

**Fixed (two layers):**
1. **Shared-secret auth** — the helper requires `Authorization: Bearer <token>`
   on every action; the token lives in `~/.hivematrix/desktopbee-token`
   (mode 600), generated by whichever side starts first and read by the daemon's
   client. A different local process without the file cannot call the helper.
2. **Server-side approval** — the helper classifies each action's tier and
   refuses any non-`free` action unless the request carries `approved: true`
   (set by the daemon's approval flow). Approval is no longer client-only.
**Verified live:** no-token → 401; free action ok; `app.activate` without
`approved` → refused server-side; with `approved` → executes.

## F3 — `script.run` allows arbitrary shell via AppleScript `do shell script`  · HIGH · FIXED

The app allowlist gated the target app name, but an arbitrary AppleScript body
could `do shell script` to run any shell command.

**Fixed:** `Scripting.run` rejects any script containing `do shell script`,
`do script`, or `system attribute` (case-insensitive) — the shell-escape paths.
script.run remains the approval tier and is gated by the app allowlist; combined
with F2 (only the daemon can call) the escape is closed.
**Verified live:** an approved, allowlisted-Finder `script.run` containing
`do shell script "whoami"` → blocked.

## F4 — "System Events" in the default script allowlist is very broad  · MEDIUM · FIXED

**Fixed:** the default `SCRIPT_APP_ALLOWLIST` is now `["Finder"]` only. System
Events (system-wide UI scripting + input synthesis) must be explicit opt-in via
the `DESKTOPBEE_SCRIPT_ALLOWLIST` env var.
**Verified live:** an approved `script.run` targeting `System Events` →
rejected (not in allowlist).

## F5 — Updater attached auth to any channel URL scheme  · LOW · FIXED

`getUpdaterConfig()` built the `Authorization: Bearer <token>` header without
checking the URL scheme; a misconfigured `channelUrl` (http / wrong host) would
have leaked the token in cleartext.

**Fixed:** the token is now attached only when `channelUrl` is `https://`.
(Current config uses `https://api.github.com/...`, so no behavior change.) A
host allowlist would harden this further.

## F6 — Broad `gh` token stored on disk  · LOW · FIXED (tooling) / operator action

**Fixed (tooling):** `scripts/set-update-token.sh` installs a fine-grained PAT —
validates it can read the repo's releases (Contents:Read), writes it to
`~/.hivematrix/keys/github-token` mode 600, replacing the broad `gh` token.
[UPDATE-CHANNEL.md](UPDATE-CHANNEL.md) documents the exact PAT scope (repository
`irvencassio/hivematrix`, Contents: Read-only). GitHub doesn't allow minting
fine-grained PATs via API/`gh`, so the operator generates it in the web UI once
and runs the script. **Operator action remaining:** mint + install the PAT.

## F7 — Hand-rolled HTTP/1.1 parser in the helper  · INFO

The helper parses HTTP requests with a minimal custom parser. Loopback-only bind
bounds exposure; malformed requests are low risk here but the parser is not
hardened against adversarial input. Acceptable given the threat model; revisit
if the helper is ever exposed beyond loopback.
