Skills & Commands Sharing

A practical map for sharing, importing, and browsing reusable skills and slash commands across Claude and ChatGPT.

Non-technical explanation

HiveMatrix has two kinds of reusable know-how. A skill is a portable recipe agents can read and follow. A command is a local slash command or folder skill that a profile runtime already knows how to run.

The catalog browser puts both in one searchable place. The important addition is a clear compatibility label, so an operator can tell whether an item is universal or tied to Claude or ChatGPT.

Command compatibility is inferred from frontmatter such as model: opus or model: gpt-5. Unknown or missing model values fall back to All so old commands remain visible.

Share out

Copy a skill

Library skills can be copied as shareable markdown from the catalog. This preserves frontmatter such as name, description, kind, and compatibility.

Publish a skill

The catalog can publish an existing reviewed skill to personal, team, org, or public scopes when the configured source and scan gates allow it. Shared skills keep provenance and can be imported by another HiveMatrix install.

Local commands

Local slash commands remain native to the profile directory. Share the command markdown or the folder skill source when another machine should run the same local command.

Import

URL or paste

Remote or pasted skill markdown imports into the brain skill library as untrusted until the operator reviews and trusts it.

Shared scopes

Configured personal, team, org, or public sources can be browsed before import. Imported shared skills are scanned and marked with trust state.

Local profile skills

Folder skills under a local profile can be bulk-imported into the brain library. The imported recipe is portable; bundled scripts stay at the source path, so native command execution remains the high-fidelity path for assets.

Catalog browser behavior

  • One catalog combines brain-library skills and local profile commands.
  • Search includes name, description, kind, and compatibility labels.
  • List rows and detail panels show compatibility chips.
  • Commands can be launched as slash-command tasks only after they are found in a fresh local catalog scan.
  • Library skills can be viewed, copied, published, trusted, deleted, and run through the existing skill routes.
